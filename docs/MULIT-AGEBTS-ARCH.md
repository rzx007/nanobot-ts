## 方案 B：跨进程 IPC 通信

```
┌──────────────────────────────────────────────────────────────────────┐
│                    多进程 IPC 通信架构                                │
└──────────────────────────────────────────────────────────────────────┘

     ┌──────────────────────────────────────────────────────────────┐
     │                    Main Process (父进程)                      │
     │                                                                │
     │  ┌────────────────────────────────────────────────────┐     │
     │  │            AgentLoop (主代理)                          │     │
     │  │                                                        │     │
     │  │  spawn(task) {                                         │     │
     │  │    const { port1, port2 } = new MessageChannel();     │     │
     │  │                                                        │     │
     │  │    const child = fork('subagent-worker.js', [], {    │     │
     │  │      stdio: ['ignore', 'ignore', 'ignore', 'ipc'],   │     │
     │  │      detached: false  // 保持父子关系                 │     │
     │  │    });                                                │     │
     │  │                                                        │     │
     │  │    // 发送 port 给子进程                                │     │
     │  │    child.send({                                        │     │
     │  │      type: 'INIT',                                    │     │
     │  │      taskId: uuid(),                                  │     │
     │  │      task,                                            │     │
     │  │      port: port2  // 传输端口                         │     │
     │  │    }, [port2]);                                       │     │
     │  │                                                        │     │
     │  │    // 监听子进程消息                                   │     │
     │  │    child.on('message', (msg) => {                     │     │
     │  │      if (msg.type === 'RESULT') {                     │     │
     │  │        await this.bus.publishInbound({               │     │
     │  │          channel: 'system',                           │     │
     │  │          content: msg.result                          │     │
     │  │        });                                            │     │
     │  │      }                                                 │     │
     │  │    });                                                 │     │
     │  │                                                        │     │
     │  │    this.runningProcesses.set(taskId, {               │     │
     │  │      child,                                           │     │
     │  │      port: port1                                     │     │
     │  │    });                                                │     │
     │  │  }                                                     │     │
     │  └───────────────────┬────────────────────────────────┘     │
     │                      │                                          │
     │                      │ fork() + IPC                             │
     │                      ▼                                          │
     └──────────────────────┼──────────────────────────────────────────┘
                           │
                           │ Node.js IPC (process.send / port.postMessage)
                           │
                           ▼
     ┌──────────────────────────────────────────────────────────────┐
     │              Subagent Worker Process (子进程)                 │
     │                                                                │
     │  ┌────────────────────────────────────────────────────┐     │
     │  │           subagent-worker.js                         │     │
     │  │                                                        │     │
     │  │  let messagePort: MessagePort;                        │     │
     │  │  let taskId: string;                                   │     │
     │  │                                                        │     │
     │  │  process.on('message', ({ type, task, port }) => {    │     │
     │  │    if (type === 'INIT') {                            │     │
     │  │      taskId = msg.taskId;                             │     │
     │  │      messagePort = port;  // 接收父进程的端口         │     │
     │  │                                                        │     │
     │  │      // 启动子代理逻辑                                   │     │
     │  │      runSubagent(task).then(result => {               │     │
     │  │        // 通过 port.send() 发送结果                     │     │
     │  │        messagePort.postMessage({                      │     │
     │  │          type: 'RESULT',                              │     │
     │  │          result,                                       │     │
     │  │          taskId                                         │     │
     │  │        });                                             │     │
     │  │      });                                                │     │
     │  │    }                                                   │     │
     │  │  });                                                   │     │
     │  └───────────────────┬────────────────────────────────┘     │
     │                      │                                          │
     │                      ▼                                          │
     │  ┌────────────────────────────────────────────────────┐     │
     │  │              Subagent Logic                         │     │
     │  │                                                        │     │
     │  │  async function runSubagent(task) {                │     │
     │  │    const agent = new SubagentAgent();               │     │
     │  │    const result = await agent.execute(task);        │     │
     │  │    return result;                                    │     │
     │  │  }                                                     │     │
     │  └────────────────────────────────────────────────────┘     │
     │                                                                │
     └────────────────────────────────────────────────────────────┘

     ┌──────────────────────────────────────────────────────────────┐
     │                    MessageChannel (IPC通道)                    │
     │                                                                │
     │     port1 (父进程)  ←──────────→  port2 (子进程)               │
     │                                                                │
     │     消息类型:                                                  │
     │     • INIT: 父进程初始化子进程                                  │
     │     • RESULT: 子进程返回结果                                   │
     │     • HEARTBEAT: 心跳检测                                     │
     │     • CANCEL: 父进程取消子进程                                  │
     │                                                                │
     └──────────────────────────────────────────────────────────────┘

流程：
1. 父进程调用 spawn()
2. fork() 创建子进程
3. 父进程通过 IPC 发送 MessageChannel 的 port2 给子进程
4. 子进程接收 port，建立双向通信
5. 子进程执行任务，通过 port.postMessage() 发送结果
6. 父进程通过 port.onmessage() 接收结果，发布到 MessageBus
7. 主代理处理系统消息，总结给用户

特点：
✅ 进程隔离，子进程崩溃不影响主进程
✅ 双向通信，可以取消子任务
✅ 使用原生 Node.js IPC，无需外部依赖
⚠️  需要序列化数据 (postMessage 使用结构化克隆)
⚠️  不能传输不可序列化的对象 (如函数、DOM)
⚠️  代码相对复杂
```
